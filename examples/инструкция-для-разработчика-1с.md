# Инструкция: интеграция 1С УПП с дашбордом продаж

## Что нужно сделать

Настроить автоматическую отправку данных о продажах и планах из 1С УПП
на сервер дашборда раз в час. Дашборд будет обновляться в реальном времени.

---

## Шаг 1 — Создать общий модуль

В конфигураторе:

**Конфигурация → Общие → Общие модули → (правая кнопка) Добавить**

Параметры модуля:
- Имя: `ДашбордПродаж`
- Флаги: ✅ Сервер, ✅ Вызов сервера

Скопировать в модуль содержимое файла:
```
examples/upp-http-export-template.bsl
```

---

## Шаг 2 — Проверить запрос к продажам

В модуле есть функция `ПолучитьПродажи()` — она обращается к:

```
РегистрНакопления.Продажи.Обороты(...)
```

Поля которые используются:
| Поле в запросе | Что это |
|---|---|
| `Прод.Склад.Код` | Код склада (точка продаж) |
| `Прод.Склад.Наименование` | Название склада |
| `Прод.Номенклатура.Код` | Код товара |
| `Прод.Номенклатура.Наименование` | Название товара |
| `Прод.Номенклатура.НоменклатурнаяГруппа.Наименование` | Категория товара |
| `Прод.КоличествоОборот` | Количество |
| `Прод.ВыручкаОборот` | Выручка (сумма со скидкой) |
| `Прод.СебестоимостьОборот` | Себестоимость |

Проверить в консоли запросов что регистр и поля существуют в данной конфигурации.

---

## Шаг 3 — Уточнить источник плана продаж

В модуле есть функция `ПолучитьПланы()` — она обращается к:

```
РегистрНакопления.ПланыПродаж.Обороты(...)
```

**Нужно уточнить**: из какого регистра/документа берётся «Идеальный результат»
в отчёте «Анализ план-факта по магазинам». Это может быть:

- `РегистрНакопления.ПланыПродаж` — стандартный регистр УПП
- `РегистрСведений.ПланыПродажПоМагазинам` — если настраивался вручную
- Другой объект конфигурации

Если регистр называется иначе — поправить запрос в функции `ПолучитьПланы()`.
Если план хранится только по магазинам без разбивки по товарам — адаптировать
запрос и структуру данных (убрать `productId`, оставить только `storeId` и `amount`).

---

## Шаг 4 — Создать регламентное задание

**Конфигурация → Общие → Регламентные задания → Добавить**

Параметры:
- Имя: `ВыгрузкаВДашбордПродаж`
- Метод: `ДашбордПродаж.ВыполнитьВыгрузкуВДашборд`
- Использование: ✅ включено

Расписание (кнопка «Расписание»):
- Повторять каждые: **3600 секунд** (1 час)
- Или настроить по времени: например каждый день в 07:00, 10:00, 13:00, 16:00, 19:00

---

## Важно

Тип `URI` в 1С доступен только в мобильном приложении и недоступен на сервере.
В модуле `ОтправитьPOST()` хост и путь прописаны явно — не использовать `Новый URI()`.

---

## Шаг 5 — Проверить подключение вручную

Перед запуском по расписанию — проверить вручную:

1. Открыть **Сервис → Консоль функций** (или любой другой способ запустить код на сервере)
2. Выполнить:
```bsl
ДашбордПродаж.ВыполнитьВыгрузкуВДашборд();
```
3. В окне сообщений должен появиться ответ вида:
```json
{"ok":true,"run":{"status":"success","period":"2026-04",...}}
```

Если ответ `status: "success"` — всё работает, данные ушли на сервер.

## Что происходит после отправки

- Данные сохраняются в PostgreSQL
- Дашборд в браузере обновляется автоматически (без перезагрузки страницы)
- Каждая отправка логируется — можно посмотреть на `/api/ingest/runs`
- Повторная отправка одного и того же пакета безопасна (дедупликация по хэшу)
